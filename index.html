How it works

Each snippet is one SSH command.

Termius fires it at your Mac's Tailscale IP. The Mac spawns a detached autopilot session, logs progress to ~/.claude/orchestrator/runs/, and fires a macOS notification when done. Never pushes. Commits locally with [autopilot] prefix.

Mac stays reachable as long as Amphetamine (or equivalent) keeps it awake with the lid closed.

Create in Termius

Once per snippet, five taps.

  1. Snippets tab → tap +.
  2. Name it (use the labels below).
  3. Paste the script. Tap the Copy button here, then ⌘V in the Script field.
  4. Save.
  5. To fire: open the snippet → Run or Send to host → pick your Ensemble Mac host.

Themed missions

Custom prompt, timed.

Each one instructs the agent to work a specific kind of task, bounded in time. Safety rails still apply — local commits only, never pushes.

Docs Polish — 15 min
prose
Pick one .md file in docs/; polish typography, fix broken links, tighten prose. One small commit.
bash ~/.claude/orchestrator/bin/mobile_dispatch.sh fire 15 "Read docs/*.md. Pick one file that would benefit most from a polish pass. Fix typography, broken internal links, grammar slips, and tighten prose surgically. Commit once with [autopilot] prefix. Never push."
README Sweep — 20 min
metadata
Refresh README.md files to reflect current commit state. Surgical, one commit per file.
bash ~/.claude/orchestrator/bin/mobile_dispatch.sh fire 20 "Read README.md at repo root, plugins/ensemble/README.md, and plugins/ensemble/bootstraps/README.md. Update claims, command examples, and file refs to match current repo state (check git log for recent changes). Surgical edits only. One commit per file. Never push."
TODO Closer — 30 min
code
Grep for TODO/FIXME; pick one; close it properly with an implementation.
bash ~/.claude/orchestrator/bin/mobile_dispatch.sh fire 30 "Grep the repo for TODO and FIXME comments (exclude .git, node_modules, research/). Pick ONE that is bounded, clear, and closable in 25 minutes. Implement the fix, remove the comment, commit with [autopilot] prefix. Log your choice + reasoning in the run log. Never push."
Cleanup Sweep — 15 min
hygiene
Trailing whitespace, obvious dead code, inconsistent formatting. No refactors, no logic changes.
bash ~/.claude/orchestrator/bin/mobile_dispatch.sh fire 15 "Scan bin/*.sh and plugins/ensemble/bin/*.sh for: trailing whitespace, obvious dead commented code (not deliberate comments), inconsistent indentation. Fix surgically. One commit. NO refactors, NO logic changes. Never push."

KILL — emergency stop

Releases the autopilot lock immediately, kills any in-flight session, drops the kill-switch signal for the Maestro's shutdown cascade. Safe to tap anytime.

Safety

Guaranteed — you don't enforce them.

No git pushPrompt explicitly forbids it. safety_constraints.md enforces at a higher level. Never included in the default prompt.
No deploysSame — prompt + safety_constraints.md.
Local commits onlyAll commits use [autopilot] prefix. Push requires explicit in-chat approval from a Maestro or Agent session later.
Hard duration ceilinggtimeout (N+2)m wrapper kills the session if it overruns. N is the minutes you fired.
Mode gateAborts cleanly if state.json.mode ≠ orchestrated.
Lock conflict gateAborts cleanly if another autopilot is already holding the lock.
Post-mortem logEvery run writes to ~/.claude/orchestrator/runs/autopilot-<stamp>.md + .raw.log.
The phone is the trigger, not the worker. Your phone just sends a ~100-byte SSH command and gets a one-line acknowledgment. All the work runs on your Mac. Battery-cheap, data-cheap, latency < 1s over Tailscale.

Troubleshooting

The three quickest fixes.

"Connection refused"Mac is asleep. Open the lid, or start an Amphetamine indefinite session before leaving home.
"Permission denied (publickey)"SSH key mismatch. In Termius: Host → edit → ensure your ensemble_phone key is selected.
"lock: HELD" foreverFire the KILL snippet. Releases the lock immediately + kills any hung autopilot.

Anything else — check ~/.claude/orchestrator/mobile_dispatch.log and the relevant runs/autopilot-*.raw.log for the exact error text.
